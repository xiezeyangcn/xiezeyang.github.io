Keymaster的SPL版本绑定

发表于 2019-03-18 | 分类于 Security

在Keymaster 1.0中，所有Keymaster密钥都会绑定至信任根或Verified Boot密钥。在Keymaster 2.0和3.0中，所有Keymaster密钥也绑定至OS及系统image的Security Patch Level(SPL)；可确保发现旧软件漏洞的人无法将手机系统还原至包含漏洞的软件版本，也无法使用在新软件中所创建的密钥；当系统升级时，密钥会随着系统一起“升级”。在Keymaster 4.0中，为适配Treble模块化结构，boot,system,vendor等多个系统分区均包含SPL信息，另外支持AVB的系统还会将这些SPL及OS版本信息存入vbmeta中，以便bootloader可以将这些内容传给Keymaster。

在实际系统开发中，经常会遇到合入新的Google Security Patch造成CTS/VTS等认证测试失败，而版本发布计划又不能推迟，最后不得不采用临时回退安全补丁方式的事情发生。这样就很容易收到测试人员反馈：半擦升级/OTA升级进入输入密码界面，分析日志不难发现，都是上述提到的SPL绑定Keymaster惹的祸。下面就来跟一下Log打印流程：
