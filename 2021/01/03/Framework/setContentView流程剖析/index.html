setContentView流程剖析

发表于 2021-01-03 | 分类于 Framework

写过app的小伙伴们都知道在Activity中需要调用setContentView方法将layout xml文件加载到Activity中。下面我们就来看看它背后的逻辑（基于 Android 12 Preview 1 版本 AOSP 源代码）.

1. Activity.setContentView

首先，来到四大组件之老大Actvity.java这里会看到，它会get window，然后再 set ContentView。接着会根据需要初始化 window decor 之 actionbar （导航栏）。

2. Window.setContentView

跟着 mWindow 这个最顶级得 window 我们来到了 Window.java, 很明显就能看到它仅仅是一个抽象类。那么问题来了，它得具体实现在哪里呢？

3. Activity.attach

为了揭秘上面这个问题，我们回到 Activity.java , 来找找看 mWindow 在哪里被赋值的。在 attach 方法中，我们能看见 mWindow 原来是 PhoneWindow 类的对象啊。

4. PhoneWindow.setContentView

很明显看到 PhoneWindow 类用于实现 Window 的了吧。那接下来正式进入正题。

5. PhoneWindow.installDecor

接下来看看调用 installDecor 是如何创建 decor view 的。

6. PhoneWindow.generateDecor

调用 generateDecor 方法初始化顶级视图-decorview。

7. PhoneWindow.generateLayout

再回过来看看 generateLayout 方法里都干了些啥。

至此，关于 setContentView 相关源码流程简述完成，做个简单的总结吧：
先创建了 DecorView 的一个对象，它是应用窗口的根视图，然后设定UI风格啥的，再将布局放到ID_ANDROID_CONTENT对应的帧布局中，为后面显示出来做好准备工作。
