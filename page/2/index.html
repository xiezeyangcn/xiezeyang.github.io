LK初始化流程

发表于 2019-01-03 | 分类于 Boot

LK的全称是litter kernel，同u-boot类似，它也是一种bootloader程序。

lk代码通常位于bootable/bootloader/lk目录，其树结构如下：
├── AndroidBoot.mk
├── app :lk中应用，如fastboot，shell等
├── arch :架构相关，如arm，x86等
├── dev :通用设备接口，如usb，lcm等
├── include :头文件
├── kernel :lk核心模块，包括thread，timer等
├── lib :–>一些c库
├── LICENSE
├── make :lk编译脚本
├── makefile
├── platform :不同平台/芯片对应的mtk驱动
├── project :不同项目的makefile文件
├── scripts :JTAG脚本
├── target :具体目标设备的定义及初始化

lk主要完成以下工作：
1.初始化硬件，设置中断向量表，初始化MMU，Cache，USB等。
2.加载boot.img，引导系统启动。
3.对不同boot模式的支持，如fastboot，recovery，download mode等。

Bootloader之二三事

发表于 2018-12-25 | 分类于 Boot

一、bootloader概述

bootloader是用来启动kernel的供应商专用映像。它会监听设备状态，并负责初始化TEE、绑定它的信任根。
bootloader可能会直接将一个新的映像烧入相应的分区中，也可能会使用recovery开始重新烧录过程（类似OTA操作过程）。
另外，bootloader会在启动kernel之前先验证boot、recovery分区的完整性，并显示相应的警告。

bootloader启动流程如下：
(1)首先加载bootloader。
(2)bootloader初始化内存。
(3)如果是A/B分区，则确定要启动的当前槽位。
(4)确定是否应改为启动recovery模式。
(5)bootloader加载映像，其中包含内核和ramdisk。
(6)bootloader开始将内核作为可自行执行的压缩二进制文件加载到内存中。
(7)内核将自身解压缩并开始执行到内存中。
(8)旧设备从ramdisk加载init，而新设备从system分区加载它。
(9)init从system分区中启动并开始装载其他所有分区，如/vendor，然后开始执行代码以启动设备。

Userdata分区加密方式

发表于 2018-11-16 | 分类于 Security

Android系统主要支持FDE/FBE两种加密方式。
Android P及更高版本支持元数据加密。该密钥受keymaster保护，而keymaster受verified boot保护。使用元数据加密时，启动出现的单个密钥会加密未通过文件级加密进行加密的任何内容。

一、 Full-disk encryption(FDE)

1.1 概述

Android L及更高版本支持全盘加密。全盘加密是使用密钥（由用户设备密码保护）对设备上用户数据进行编码的过程。设备加密后，所有由用户创建的数据在存入磁盘前都会字典加密，且所有读取操作都会在将数据返回给调用进程之前自动解密数据。用户必须提供credentials才能访问磁盘。

全盘加密采用128位AES加密算法，支持‘默认’，‘PIN码’，‘密码’，‘Pattern图案解锁’四种加密状态。首次启动，会先随机生成128位主密钥，然后使用默认密码’default_password’（cryptfs.cpp）和存储的salt对其进行hash处理，接着由TEE生成hash签名以加密该主密钥。（16kb）

快速加密方式：只对userdata分区已使用的分块进行加密，避免首次开机耗时过长。目前仅ext4和f2fs类型文件系统支持该快速加密方式。

加密过程由init/vold进程管理。init负责调用vold，然后vold来设置相关属性以触发init中事件。系统会使用vdc的cryptfs命令来调用vold中的加密功能。主要命令有：checkpw、restart、enablecrypto、changepw、cryptocomplete、verifypw、setfield、getfield、mountdefaultencrypted、getpwtype、getpw、clearpw等。

无论加解密还是清除data分区过程，data分区都不能处于挂载状态。所以为了确保framework启动，data分区就需要挂载一个临时文件系统，使得android可提示输入密码以清除数据此类建议。

在实际工作中可能会遇到一些加解密错误，比如密码匹配无法解密，用户输错密码此次达到30次。若这些错误未解决，则会提示用户清除数据恢复出厂设置。
