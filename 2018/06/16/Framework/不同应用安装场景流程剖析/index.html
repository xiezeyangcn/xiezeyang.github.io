不同应用安装场景流程剖析

发表于 2018-06-16 | 分类于 Framework

应用安装主要分为以下四类场景：

(1)安装器安装：调用PackageInstallActivity的startInstall方法，点击OK按钮后进入PackageManagerService完成拷贝解析安装工作；

(2)adb命令安装：通过pm参数，调用PM的runInstall方法，进入PackageManagerService安装安装工作；

(3)应用市场静默安装：下载完应用后通过AccrssibilityService监控界面，自动点击安装；

(4)系统预置应用安装：系统启动时，通过SystemServer进入PackageManagerService完成目录扫描解析安装工作。

其中系统预置应用安装，主要分两个部分，前一阶段可参阅之前的PackageManagerService启动过程中介绍，后一阶段同其它三种场景中最后的安装过程大同小异，将在后续博文中详细介绍。本文将重点介绍前三种安装场景。

一、安装器安装

1.1 InstallStart

主要工作：

(1)判断设置中“未知来源”选项是否勾选，若未勾选，则转入安装未知来源项界面；

(2)对于Android 8.0及以上版本，会先检查是否申请安装权限，若没有则中断安装；

(3)根据uri判断，若是file或者content则调用InstallStaging；若是package则调用PackageInstallerActivity。

1.2 PackageInstallerActivity

主要工作：

(1)显示确认安装界面；

(2)检查是否允许、初始化安装过程;

(3)在准备安装的之前，检查应用列表判断该应用是否已安装，若已安装则提示该应用已安装，由用户决定是否替换；

(4)在安装界面，显示该应用运行时将要申请的权限；

(5)点击OK按钮确认安装后，会调用InstallInstalling开始安装工作。

1.3 InstallInstalling

主要工作：

(1)检查package.list以确认该应用是否已存在；若不存在则调用PackageParser来解析应用；

(2)调用PackageInstaller创建会话；

(3)onResume时创建同步栈，打开安装session，设置安装进度条，开始进入frmamework安装过程。

二、adb命令安装

2.1 cmmandline

主要工作：

判断adb指令中是否包含install字段，若包含则调用install_app。

2.2 adb_install

主要工作：

(1)解析指令参数，若包含-r则删除重新安装；

(2)判断安装模式，若有设置legacy则默认为push模式，否则为stream模式。

(3)判断参数个数小于2，则报错；

(4)根据安装模式调用install_app_legacy或install_app_streamed；

(5)判断参数中是否包含apk/apkx后缀文件，若不包含则报错；

(6)将apk拷贝至/data/local/tmp目录；

(7)调用pm指令（andrid p之后替换为cmd package指令），进入framework安装。

2.3 pm

主要工作(android 9.0开始弃用)：

(1)获取getPackageInstaller；

(2)创建InstallParams对象，开始创建安装会话；

(3)开始进入安装过程。

2.4 PackageManagerShellCommand

主要工作：

(1)检查指令中是否包含”install”字段，包含则调用runInstall方法；

(2)创建InstallParams对象，开始安装过程。

三、应用市场静默安装

关于静默安装主要有五种方式：

(1)su命令获取root权限，执行pm命令。鉴于新系统的安全机制增强，不推荐使用。
参考链接：https://blog.csdn.net/guolin_blog/article/details/47803149

(2)打开设置中无障碍服务，自动完成安装过程。
餐卡链接：https://blog.csdn.net/guolin_blog/article/details/47803149

(3)加上系统签名，声明INSTALL_PACKAGE权限，执行pm指令。过于粗暴，不推荐使用。
参考链接：https://blog.csdn.net/tabactivity/article/details/10721521

(4)导入IPackageManager等相关代码，加上系统签名，声明INSTALL_PACKAGES权限。
参考链接：https://www.jianshu.com/p/326ea728e5d3

(5)适用于Android P之后，调用PackageInstall相关代码，加上系统签名。
参考链接：https://www.jianshu.com/p/54d6cbc5e689
